Выводы:

В ходе выполнения лабораторной работы были изучены и применены методы автоматического развертывания статического сайта, созданного на основе генератора документации MkDocs. Был реализован CI/CD-сценарий для двух платформ: GitHub с использованием GitHub Actions и Sourcecraft со встроенными средствами автоматизации.

В процессе работы был создан локальный git-репозиторий, к которому были подключены два удалённых репозитория (origin и sourcecraft). Это позволило организовать одновременную публикацию проекта на двух независимых платформах. Были освоены команды работы с несколькими remote-репозиториями, а также механизмы автоматического запуска сборки после выполнения git push.

Для GitHub был настроен workflow GitHub Actions, автоматически выполняющий сборку и публикацию сайта в ветку gh-pages. Дополнительно были изучены настройки GitHub Pages и работа с секретами (GITHUB_TOKEN) для автоматического деплоя.

Для Sourcecraft был реализован pipeline, выполняющий установку зависимостей, сборку сайта средствами MkDocs и публикацию результата через отдельную release-ветку. Также были настроены параметры публикации Pages внутри Sourcecraft.

В результате работы были получены практические навыки:

настройки CI/CD для статических сайтов; работы с несколькими удалёнными git-репозиториями; автоматической публикации документации; настройки GitHub Pages и Sourcecraft Pages; организации структуры проекта для автоматической сборки и деплоя.

Итогом выполнения лабораторной работы стал успешно развернутый статический сайт, доступный одновременно через GitHub Pages и Sourcecraft Pages с автоматическим обновлением после отправки изменений в репозиторий.
